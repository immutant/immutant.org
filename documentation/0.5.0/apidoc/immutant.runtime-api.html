immutant.runtime

This namespace is solely for use during the application runtime
bootstrapping process. Applications shouldn't use anything here.

initialize

function
Usage: (initialize init-fn config-hash)

Attempts to initialize the app by calling an init-fn (if given) or, lacking that,
tries to load an immutant.clj from the app-root. In either case, post-initialize is called
to finalize initialization.
Source

require-and-invoke

function
Usage: (require-and-invoke namespaced-fn & [args])

Takes a string of the form "namespace/fn", requires the namespace, then invokes fn
Source

set-app-config

function
Usage: (set-app-config config project)

Takes the full application config and project map as data strings and makes them available as data under the :config and :project keys in the registry.
Source

immutant.runtime.bootstrap

Functions used in app bootstrapping. Should not be used in an app runtime.

get-dependencies

function
Usage: (get-dependencies app-root profiles resolve-deps?)
       (get-dependencies project resolve-deps?)

Resolves the dependencies for an application. It concats bundled jars with any aether resolved
dependencies, with bundled jars taking precendence. If resolve-deps is false, dependencies aren't
resolved via aether and only bundled jars are returned.
Source

read-and-stringify-full-app-config

function

Loads the full app config and stringifies the keys.
Source

read-full-app-config

function
Usage: (read-full-app-config descriptor-file app-root)

Returns the full configuration for an app. This consists of the :immutant map
from project.clj (if any) with the contents of the descriptor map merged onto it (if any). Returns
nil if neither are available.
Source

read-full-app-config-to-string

function
Usage: (read-full-app-config-to-string descriptor-file app-root)

Returns the full configuration for an app as a pr string that can move across runtimes.
Source
